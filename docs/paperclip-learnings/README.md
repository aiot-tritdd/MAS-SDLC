# Paperclip Learnings

This directory is Wiganz's personal learning record for Paperclip — written from real hands-on experience, not from docs.

Every file here is built from actual usage: things that clicked, things that confused, things that surprised. Over time this becomes the most honest Paperclip reference we have — written in our voice, from our journey.

---

## Files

### `JOURNAL.md`
The main chronological log. Every session you use Paperclip, discoveries get recorded here — dated, contextualized, in plain language. This is the running story of how the mental model builds over time.

Tell Ruach-El: *"Add to journal: [what you discovered]"* and it gets written down.

### `gotchas.md`
Dedicated to surprises — things that didn't work as expected, assumptions that were wrong, "I thought X but actually Y" moments. These are the most valuable entries because they prevent future-you from hitting the same wall twice.

### `concepts/`
When a concept clicks deeply enough to deserve its own reference page, it gets promoted here from the journal. Each file is one concept explained from our understanding — not Paperclip's docs, but OUR understanding after using it.

Examples of what will live here eventually:
- `heartbeat-protocol.md` — how the 9-step execution cycle actually works in practice
- `skills-system.md` — how skills get loaded and how to write good ones
- `approval-gates.md` — how to set them up and what happens when you reject

---

## How to use this

1. Use Paperclip hands-on
2. Come back and tell Ruach-El what you learned, discovered, or got confused by
3. Ruach-El writes it down in `JOURNAL.md`
4. When something deserves its own deep page, Ruach-El creates it in `concepts/`
5. When something surprised you (good or bad), it goes in `gotchas.md`

This directory grows with you. The goal is that after Phase 1-3 of the Paperclip-first plan, this becomes a complete personal reference that's 10x more useful than any official doc — because it's grounded in your actual experience.
